[AOP, 관점지향 프로그래밍]
    1) 정의
        핵심 로직과 공통 로직을 분리하여 관리하는 프로그래밍 방식
        부가 기능을 하나로 모듈화하여 핵심 비즈니스 로직을 분리
         >> 유지보수 및 확장성
    2) 사용처
        로그(기록), 트랜잭션, 보안(인증/권한)
    3) 설치
        ` implementation 'org.springframework.boot:spring-boot-starter-aop' `

    4) 사용
        (1) AOP class 어노테이션 주입
            ```
                @Aspect         // 스프링 AOP container에 등록
                @Component      // 스프링 bean container 등록
            ```
        (2) AOP 메소드 작성 및 어노테이션 주입
            @Before         : Service 메소드 이전에 AOP 메소드가 실행           : 권한 검사
            @After          : Service 메소드 이후에 AOP 메소드가 실행           : 로그 용도, 연동 해제
            @AfterReturning : Service 메소드 결과값을 받아올 수 있는 어노테이션   : 상기 동일
            @Around         : 메소드 실행 시기를 직접 컨트롤 할 수 있음           : 성능평가, 트랜잭션
            ```
                @Before()
                public void check1(){
                    System.out.println("[코로나] 열 체크");
                }
            ```
        (3) @Before / @After 어노테이션에 조건 기재
            ex) ` @Before("execution( * AopService.*(..) ) ") `
                ` @Before("execution( ① ②.③ ( ④ ) ) ") `

            ① ③메소드의 리턴 타입
                * : 모든 리턴 타입
                int : int 리턴 타입
            ② Service 또는 AOP 대상 클래스 경로
                - 동일 패키지 : 클래스명만 기재
                - 다른 패키지 : Java 패키지 이하의 경로
            ③ .메소드명
                - ②에 해당하는 class 내의 대상 메소드
            ④ 매개변수
                - 메소드에 어떠한 매개변수를 포함할 때 실행할 것인지
                - (..) : 모든 매개변수에 대하여

            추가!!  `&& args(⑤)`
            ⑤ 매개변수 인자값을 바인딩할 이름 정의
            `execution( boolean Service.method(String, int) ) && args(name, no)`

            overload로 인하여 동일한 메소드 명에 다른 반환/매개변수로 구분하기 위해

        (4) @AfterReturning
            @AfterReturning( value = " " ,returning = "result")
            - value 에는 execution 조건절을 입력
            - result에 결과값으로 바인딩할 변수명 을 정의

        (5) @Around
            ```
                @Around("execution ( * class.method (..))")
                public Object AOP메소드명( ProceedingJoinPoint joinPoint ){

                    joinPoint                   : AOP 객체
                    joinPoint.getSignature()    : AOP 메소드를 실행하는 비즈니스 메소드 명을 반환
                    joinPoint.getArgs()         : 실행메소드의 매개변수를 배열 타입으로 확인
                    joinPoint.proceed()         : Service 메소드 실행하기, 예외처리 필수
                    Object result = joinPoint.proceed();    : Service 메소드 실행 + 결과 반환 받기 = returning
                    return result;              : Service 실행 결과 Object를 결과로 반환
                }
            ```

    5) AOP를 사용하는 주요 어노테이션
        (1) @Transactional
            - JDBC에서 SQL 시작과 끝을 하나의 로직으로 만드는 방법
            - 일반적으로 Service 에서 사용


        (2) @Log4j2
            - 로그처리
        (3) 스프링 시큐리티 (난이도 높음)
